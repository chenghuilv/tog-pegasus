WBEM Provider Data Sheet

Operating System Provider

This data sheet provides brief, very basic and the most update to date information available about the generic Operating System WBEM provider (implemented using the Pegasus C++ API). This provider supports the CIM_OperatingSystem class, as well as a PG_OperatingSystem subclass that adds the properties SystemUpTime and OperatingSystemCapability.  This document is not intended to replace the manual or man pages.   The intended audience of this document is client/provider developers and sophisticated end-users who want to use this provider. This data sheet was last updated: 22-Apr-02.

1. Provider Overview

·        Description
This generic Operating System provider implements a CIM_OperatingSystem and a PG_OperatingSystem subclass of CIM_OperatingSystem (adding the properties of SystemUpTime and OperatingSystemCapability).  SystemUpTime is a convenience property, enabling direct access to this value, versus having clients/providers calculate the value from LastBootUpTime and the LocalDateTime.  OperatingSystemCapability exposes if the OS itself is 32-bit or 64-bit capable. This provider is for use by clients as part of a basic understanding of the identity of the Managed System on which this provider is running (typically a server or appliance).

·        Requirements
Implemented and tested on HP-UX and Debian Linux OS.  Also validated compilation on Windows.   Linux mechanism for getting maximum number of processes will only work on Linux kernels 2.4.* or later.  HP-UX mechanism for determining the capability of the OS will work on versions 11.0 and later.

·        Release History
26-Feb-02 - initial release of provider for Linux_OperatingSystem (via submittal to Pegasus open source community)

19-Mar-02 - submittal of generic OS provider with HP-UX specific code paths and - Windows paths to return OS version, type, bootup time, time, timezone, and memory information.

22-Apr-02 - Added Linux-specific code paths from Linux_OperatingSystem provider. Added support for additional properties for HP-UX and Windows, such as Caption, Description, Status for both and LicensedUsers for HP-UX.

·        Supported Managed Resources
Managed systems (servers, appliances, desktops) running HP-UX, Linux, and Windows.

·        Special Requirements & Dependencies
Linux mechanism for getting maximum number of processes will only work on Linux kernels 2.4.* or later.

2. Setting Up This Provider

·        Installing This Provider
This generic Operating System provider is available from the Pegasus open source site (CVS tree location: /pegasus/src/ManagedSystem/OperatingSystem for the makefile, headers and source, /pegasus/Schemas/pegasus/ManagedSystem/VER20/PG_OperatingSystem20.mof for the MOF file for the schema, and PG_OperatingSystem20R.mof for the provider registration). 

·        The provider executable is "libOSProvider.sl", called "OSProvider" in the registration MOF (to differentiate it from the Pegasus OS provider in the Generic directory, else provider registration results in confusion).  Provider registration current supports /root/cimv2 as an instance provider for all properties (method provider capabilities currently disabled).  No special installation instructions (will be in pegasus/build/lib/libOSProvider.sl). The OS security mechanisms are (will be) used to authorize invokation of methods such as shutdown and reboot (e.g., must be root) root.

·        Configuring This Provider
There is currently no configuration specific to this provider.

·        Future provider plans
All properties are currently read-only.  Could add support to change appropriate values (probably via modifyInstance).  Support for createInstance and deleteInstance seems undesirable for running OS instance (but may be helpful for installed OSs).  Need to enable shutdown and reboot operations when have appropriate security support.
Support for RunningOS and InstalledOS associations would be good (although RunningOS is implicit with the first instance implemented).

5. Known Defects And Performance Considerations

Linux-specific mechanism for getting maximum number of processes will only work on Linux kernels 2.4.* or later.

Reboot and Shutdown methods have been disabled for HP-UX; will be enabled in the future.
