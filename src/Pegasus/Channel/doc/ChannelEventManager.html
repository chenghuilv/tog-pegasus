Documentation

The ChannelEventManager class manages input and output events for the channel implementations. The implementations register with it so that they be notified of activity on their channels. For example, a TCP implementation needs to be notified when sockets become read enabled or write enabled. This is the least clearly defined part of the system and will be subject to some rework.
void run(Uint32 milliseconds = INIFINITE)
Waits for an event for a certain number of milliseconds and then returns. If an event occurs, then the event is dispatched. If the milliseconds argument is equal to zero, then events are checked for and dispatched and the routine immediately returns. If milliseconds is equal to INIFINITE then, the routine never returns but continues waiting for and dispatching events. If milliseconds is neither 0 nor INIFINITE, then the routine waits for the number of milliseconds specified.

This class has no child classes.
