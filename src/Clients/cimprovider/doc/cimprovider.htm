cimprovider

Name

cimprovider - disable, enable, remove or list registered providers or provider modules.

Synopsis

cimprovider -d -m module

cimprovider -e -m module

cimprovider -r -m module [ -p provider ]

cimprovider -l [ -m module ]

Remarks

           This command can be executed by privileged user(s) only.
 
Limitations
This command can be executed to disable/enable/remove one provider module or provider at a time.

Description

The cimprovider command provides a command line interface to upgrade CIM providers. In order to use cimprovider command, the specified provider module or provider has to be registered through pegasus provider registration mechanism.

The first form of cimprovider allows to place the specified provider module in the disbaled state. When a specified provider module is in the disbaled state, any new requests to the providers which are contained in the specified provider module will be blocked.

The second form of cimprovider allows to place the providers which are contained in the specified provider module back to the service from the disabled state. Now, the providers which are contained in the specified provider module is ready to accept new requestes.

The third form of cimprovider allows to remove all the instances which have same provider module name as the specified provider module or all the instances which have same provider module name and provider name as specified.

The last form of this command allows for all the registered provider modules to be listed or all the providers which are in the specified provider module to be listed.

Specifying no options with the cimprovider command will display the command usage.

Options

The cimprovider command recognizes the following options:
-d -m                     Indicates that a CIM provider module is in the disable state.

-e -m                     Indicates that a CIM provider module is in the service from the disabled state.

-r -m                     Indicates that all the instances which contains the specified provider module are to be removed.

-r -m -p               Indicates that all the instances which contains the specified provider module and provider are to be removed.

-l                          Displays all the registered provider modules.

-l -m                     Displays all the registered providers which are in the specified provider module.

Return Values

cimprovider returns one of the following values:
0         Successful completion.

1         Error

When an error occurs, an explanatory error message is written to stderr and an error value 1 is returned.

Examples

The following command places the service of provider module "OperatingSystemProvider" in the disabled state.
cimprovider -d -m OperatingSystemProvider
The following command places "OperatingSystemProvider" provider module back to the service from the disabled state.
cimprovider -e -m OperatingSystemProvider
The following command removes all the instances which contains "OperatingSystemProvider" provider module name.
cimprovider -r -m OperatingSystemProvider
The following command removes all the instances which contains "OperatingSystemProvider" provider module name and "PG_OperatingSystemProvider" provider name.
cimprovider -r -m OperatingSystemProvider -p PG_OperatingSystemProvider
The following command displays the list of registered provider modules.
cimprovider -l
The following command displays the list of registered providers which are in the "OperatingSystemProvider" provider module.
cimprovider -l -m OperatingSystemProvider
